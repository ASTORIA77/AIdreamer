写作格式

笔尖上的简洁之道

Markdown · 从语法之美到 Agent 时代的写作语言

写这个博客之前，我记 AI 笔记已经在 Obsidian 里写了很久 Markdown。不是专门学的——是为了学 AI 顺手记笔记，手指头慢慢就熟了。现在回头看，Markdown 这件事本身也值得写清楚：为什么它值得会，以及它在 Agent 时代意味着什么。

一、何为 Markdown

Markdown 是 2004 年由 John Gruber 与 Aaron Swartz 共同创立的轻量级标记语言。它的初衷只有一个——让作者专注于内容本身，而非格式的排布。

它不像 Word 那样将文字与样式纠缠在一起，也不像 HTML 那样需要繁复的标签。Markdown 用几个最常见的符号——#、*、-、>——就能表达标题、强调、列表、引用等几乎所有写作所需的结构。

三个关键特征

纯文本——任何编辑器都能打开，永不过时，永不锁定

可读性强——即便不渲染，源文本依然清晰悦目

易于转换——可一键导出为 HTML、PDF、Word、PPT 等任意格式

同一段文字的三种写法对比

Word 文档：  [需打开软件，点击 B 按钮加粗，再点 H1 设标题……]
HTML：       <h1>标题</h1> <p>这是<strong>粗体</strong>文字</p>
Markdown：   # 标题
            这是 **粗体** 文字

同一份内容，Markdown 用最少的符号，传递最清晰的结构。这正是它二十年来历久弥新的理由。

二、为何 Markdown 在今天尤为重要

当我们走入 Agent 时代，Markdown 已不只是一种格式，而是人类与机器对话的"通用语"。

它是程序员、写作者、研究者共同的母语

从 GitHub 的 README，到 Notion、Obsidian、语雀的笔记；从博客 Hexo、Jekyll，到论文 Pandoc 转换——Markdown 是当今最广泛被采纳的写作格式。学会它，意味着你可以用同一套语法，在几乎所有现代写作平台上无缝迁移。

在 AI Agent 时代，Markdown 是"机器可读的优雅文字"

这是我最想强调的一点——Markdown 正在成为人与 AI 之间最高效的沟通语言。

①
大模型原生输出 Markdown。无论是 ChatGPT、Claude 还是各类国产大模型，它们的回答几乎全部以 Markdown 形式生成——标题、列表、代码块、表格皆然。理解 Markdown 等于理解了 AI "说话"的方式。
②
Agent 系统的提示词多用 Markdown 组织。当你写一段结构清晰的 Markdown 提示词，Agent 会更准确地理解你的意图。层级标题、有序列表、代码块边界——都是给 AI 的"语义路标"。
③
Agent 的"记忆"和"知识库"以 Markdown 存储。我在前面的文章里写过 Claude Code 的安全问题，但抛开那个不谈，主流 Agent 的 MEMORY.md、SKILL.md、README.md 全部使用 Markdown。它既能被人类读懂，又能被机器精确解析——这是 JSON 与自然语言之间最优雅的折中。
④
结构化 = 可被 AI 索引、检索、推理。当你用 Markdown 写笔记，AI 可以轻松定位到"第二章第三节"；当你用纯文本，它只能模糊地猜。结构本身就是一种向 AI 友好的写作礼仪。

它让"写作"成为一种可积累的资产

Markdown 文件是纯文本，可以用 Git 版本控制，可以同步到任意云端，可以二十年后依然打开。它让你的每一份笔记、每一篇思考，都成为可迁移、可复用、可被未来自己继承的知识资产。

当格式不再是负担，思想才能真正自由地流淌。

三、如何写好 Markdown

先掌握五个核心语法，其余皆水到渠成。

基础语法

① 标题

# 一级标题
## 二级标题
### 三级标题

# 数量代表层级，井号后必须有空格。一篇文章建议只有一个一级标题。

② 强调

**粗体**    *斜体*    ***粗斜体***    ~~删除线~~

③ 列表

- 无序列表（横线 + 空格）
- 第二项
  - 缩进两格即嵌套

1. 有序列表
2. 自动编号

- [x] 已完成的任务
- [ ] 待办任务

④ 链接与图片

[链接文字](https://example.com)
![图片描述](path/to/image.png)

⑤ 代码

行内代码用反引号：`npm install`

代码块用三个反引号 + 语言名：
```python
def hello():
    print("Hello, Markdown")
```

⑥ 引用与分隔线

> 这是一段引用，
> 可以多行。

---  ← 三个连字符即分隔线

进阶语法

表格

| 姓名 | 年龄 | 城市 |
|------|:---:|-----:|
| 小明 |  25 | 北京 |
| 小红 |  30 | 上海 |

对齐方式：:--- 左对齐 · :---: 居中 · ---: 右对齐

数学公式（LaTeX）

行内公式：$E = mc^2$
块级公式：$$ \int_a^b f(x)\ dx = F(b) - F(a) $$

折叠块

<details>
<summary>点击展开</summary>
隐藏的内容写在这里。
</details>

脚注

这是一句话[^1]。
[^1]: 这是脚注内容。

推荐编辑器

Typora

所见即所得，写作如行云流水。最适合长文与日常笔记。

Obsidian

本地存储 + 双向链接，构建你的"第二大脑"。免费且强大。

VS Code

配合 Markdown All in One 插件，是技术写作者的不二之选。

Mark Text

开源、跨平台、轻盈优雅。极简主义者的最爱。

语雀 / Notion

在线协作，团队共享，云端同步。适合团队文档。

GitHub / GitLab

在浏览器中直接编辑预览，README 与文档的天然主场。

写作工作流建议

1. 先列大纲，再填血肉。用标题层级搭起骨架，思路自然清晰。

2. 多用列表与表格。结构化的内容，比连绵的段落更易吸收。

3. 关键词加粗，让读者一眼抓住重点。

4. 代码与命令必用代码块。避免被自动格式化破坏。

5. 用 Git 管理长期写作。每一次修改都是一次思想的快照。

当你下一次与 AI 对话时，试着用 Markdown 写你的提示词——用标题分主题，用列表列要求，用代码块标示输入。你会惊讶地发现，AI 的回答变得更精准，更有条理。

Markdown 从来不只是写给人看的。它正在成为人类与机器共同的母语——一种简朴、清晰、且永远不会过时的写作方式。
