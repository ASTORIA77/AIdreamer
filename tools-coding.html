PROGRAMMING

编程工具

从面向自然语言到面向正经代码

AI 编程工具现在大致分两派：老一辈的传统编辑器 + AI 插件（如 VS Code + Copilot / 通义灵码），和新一代的原生 AI IDE（如 TRAE、Qoder、CodeBuddy、ZCode）。前者像给你配了个副驾，后者更像你把车钥匙直接给了司机。各有各的用处。

亲测过

VS Code + AI 插件

主力在用

最近的心头好。不是因为功能多强，是心态——在 miniconda 配好虚拟环境，终端敲下 code .，VS Code 在项目目录里打开的那一刻，感觉自己真的在正规化学 AI。

VS Code 本身不是 AI 工具，但装上 AI 插件之后——比如 GitHub Copilot（代码补全鼻祖，虽然现在有点被后来者赶超）、通义灵码（阿里出品，中文理解好）——编码体验就完全不同了。写注释描述功能，AI 补充代码；选中一段逻辑，AI 解释给你听。它不是帮你写代码，是帮你理解代码。

对我来说，VS Code 代表一种信号：我不再是随便玩玩，我要系统性地学了。这个心态价值大于一切功能。

TRAE（字节跳动）

用过 · 已切 VS Code

我第一个正经用的 AI 编程工具。字节跳动出的，底层基于 VS Code 架构，完美兼容 VS Code 插件生态——所以你从 TRAE 切到 VS Code 几乎零迁移成本。

TRAE 核心是两种模式：IDE 模式是你正常写代码，AI 在旁边辅助补全和对话；SOLO 模式就厉害了——你只需要用自然语言描述需求，比如「帮我写一个 Flask 的增删改查接口」，它自己规划、写代码、跑调试，你只要当甲方就行。说是「面向自然语言编程」毫不夸张。

非常适合新手入门。你不需要懂语法，就能先看到代码是怎么跑起来的。对我来说，它是帮我把「编程恐惧」这个心理门槛打掉的第一块砖。后来转向 VS Code 不是因为它不好，是因为我想更正规地写代码了。

知道但没用过

CodeBuddy · Qoder · ZCode

关注中

这三个是原生 AI IDE 里的代表选手。

Qoder（阿里）走的是「智能体自主开发」路线——你把一个模块级别的任务交给它，它自己拆子任务、查文档、写代码。跟 TRAE 的 SOLO 有点像，但 Qoder 更强调全自动，不需要你坐在电脑前跟它交互。

CodeBuddy（腾讯云）亮点是 MCP 生态集成——它不只是一个编辑器，还能连通外部工具和数据源。我说过我用 WorkBuddy，底层就是 CodeBuddy 的能力。

ZCode（智谱）是 GLM 模型官方适配的开发工具，2026 年 6 月刚出 3.0 版，用了自研 Agent 内核，国产工具里少有的「不用第三方 Agent 框架」的产品。

这三个我都还没上手——目前 VS Code 够用，再切工具成本太高。但不妨碍先记下来，以后有需求了再试。

CLI AI 工具 / Copilot

还没到这个段位

据说正经程序员除了 IDE，还会用命令行里的 AI 工具——比如在终端里直接让 AI 生成脚本、调试错误、操作 Git。像 Claude Code、OpenAI Codex CLI 就是做这个的。

至于 GitHub Copilot，可以说是代码自动补灵的鼻祖——你在编辑器里写函数名，它猜你想干嘛，自动生成整段代码。曾经在 2023-2025 年火得不得了，但现在各路 AI IDE 全都自带这个能力了，Copilot 的光环就没那么强了。

这两个品类我现在都还没接触到——不是它们不好，是我还没写足够多的代码，没有「被重复劳动逼疯」的体验，自然也不需要这些工具来救命。等以后干活干到吐了，自然就上道了。
