## 一、这个项目是什么

一个**个人 AI 学习博客**，作者 AIdreamer 以「用 AI 学 AI」为核心理念，记录学习 AI 的真实旅程。

- **不是教程站**，不是课程，不是权威指南
- **只记录一件事**：一个普通人怎么用 AI 学 AI，怎么在用的过程中越来越懂 AI
- **文章生成方式**：作者口述感悟 → 与 AI 对话梳理 → 生成 Markdown → 作者确认。想法是作者的，文字是 AI 帮忙组织的
- **写作理念**：信奉费曼学习法——能用大白话讲清楚才算真懂。**有了技术体感才写，不为填满而写**。有些主题空着是诚实的表现，不是缺陷

### 结构：星河入口 + 多主题

博客采用「星河」入口页（深色宇宙 + 莫兰迪星球）连接多个独立主题，每个主题有自己的页面和文章列表：

| 星球 | 主题 | 状态 | 配色 | 徽章色 |
|------|------|------|------|--------|
| 从小白玩转 AI Agent | 一个普通人用 Agent 学 AI 的真实记录 | **23 篇已写** | 暖驼 | `#c9a876` |

## 四、必须遵守的规矩

### 1. 视觉风格 = 莫兰迪淡雅

**这是作者最强烈的偏好，每次做 HTML/CSS 都必须遵循。**

博客主题页（浅色）：
- 米白背景 `#f5f3ef`
- 雾蓝主色 `#6b8cae` / 浅雾蓝 `#a8bdd1`
- 暖驼强调 `#c9a876` / 浅暖驼 `#ddc9a8`
- 淡绿 `#8fb89c` / 淡紫 `#a89bc4`
- 主文字 `#3d3a35`（深褐灰，不用纯黑）
- 阴影极淡（opacity 不超过 0.10）

星河入口页（深色宇宙）：
- 背景渐变 `linear-gradient(160deg, #1a2030 → #232a3d)`
- 主文字 `#d8d4cc` / 标题 `#f0ebe0`
- 强调色用浅雾蓝 `#a8bdd1`（深色背景上不要用暖驼，显脏）

**禁忌**：高饱和度亮色、深黑+高对比科技感配色、纯黑文字、过重的阴影、过大的圆角（不超过 16px）

### 2. 字体

```css
--font-serif: 'Noto Serif SC', Georgia, serif;
--font-sans: 'Noto Sans SC', -apple-system, sans-serif;
--font-mono: 'JetBrains Mono', Consolas, monospace;
```

### 3. 文章 Markdown 不写 h1 标题

标题由 `js/articles.js` 的 `title` 字段统一渲染。Markdown 里再写 `# 标题` 会导致重复。

### 4. 每次改 CSS/JS 必须递增版本号

在 HTML 引用处把 `?v=20260713` 改成当天日期。

### 5. 文章排序 = 时间正序

`ARTICLES` 数组按时间正序排列，最新文章在末尾。

### 6. 文章标签每篇 3-4 个

第 1 个恒为 `AI Agent`（主题归属），第 2-3 个是分类和核心概念。核心概念必须是读者会用来找这篇文的词。颜色按父大类同色，定义在 `js/app.js` 的 `tagClass()`。

### 7. 深色宇宙背景强调色用浅雾蓝 `#a8bdd1`

暖驼在深色背景上显脏，只用于浅色主题页。

### 8. 外部内容搬来要加时效性声明

课件、视频、他人文章搬成博客文章时，开头说明来源、制作时间、哪些已更新、可能过时。

### 9. 标题徽章（title-mark）颜色与主题一致

6 个主题各有独立徽章颜色，子页面继承同色。使用内联 `background:linear-gradient(...)`：
- Agent「玩」→ 暖驼 `#c9a876,#d4b68a`
- History「史」→ 深雾蓝 `#7a9a9a,#95b0ae`
- Complexity「杂」→ 淡绿 `#8fb89c,#a8c9b4`
- Basics「基」→ 淡紫 `#a89bc4,#bbaad0`
- Spillover「溢」→ 浅暖驼 `#ddc9a8,#e8d4b8`
- Tools「具」→ 浅雾蓝 `#6b8cae,#8fa5c9`

### 10. 工具分类页的卡片图标统一用 SVG 线条

不用系统 emoji。SVG 规格：`viewBox="0 0 24 24", stroke="#c9a876", stroke-width="1.5", fill="none"`。CSS `.hw-icon` 用 `width:22px; height:22px`（非 `font-size`）。

### 11. 工具分类页卡片左侧色条用柔化莫兰迪色

手动 HEX，不用 CSS 变量。当前配色：
- blue `#9eb8cd→#c7d7e6` / green `#b2c9b6→#d2dfd5` / purple `#b8adc6→#d2c9de` / camel `#d6bc92→#e4d4b8` / gray `#c4bbb0→#ddd6cc`

### 12. 不要编造作者的经历和计划

只写作者口述过的事实。硬件型号、个人体验、"接下来打算"——作者不说的就不写。

## 五、常见工作流

### 添加新文章（AI Agent 主题）

1. 在 `posts/` 下新建 `.md` 文件（**不要写 `# 标题`**）
2. 在 `js/articles.js` 的 `ARTICLES` 数组末尾添加配置
3. 运行 `python build.py`
4. 递增 `agent.html` 和 `index.html` 里 `articles.js` 和 `content.js` 的版本号

```js
{
  slug: 'my-new-post',
  title: '文章标题',
  subtitle: '副标题（可选）',
  date: '2026-07-04',
  tags: ['AI Agent', '概念辨析', '核心概念'],
  excerpt: '摘要',
  readTime: '约 8 分钟',
  file: 'posts/XX-my-new-post.md',
  accent: 'camel'  // blue/green/purple/camel
}
```

### 添加历史新幕

1. 在 `history-posts/` 下新建 `.md` 文件
2. 在 `js/history.js` 更新对应幕的配置
3. 运行 `python build_history.py`
4. 递增 `history.html` 里相关 JS 的版本号

### 口述转文章

1. 倾听作者口述，记录关键观点和例子
2. 梳理结构（标题 + 副标题 + 2-5 小节）
3. 生成 Markdown（风格：直率、有自嘲、用具体场景说话）
4. 交给作者确认
5. **不要自行添加作者没说过的观点**

### 修改 CSS/JS

1. 改前先读 `~/.workbuddy/skills/morandi-blog-studio/references/pitfalls.md`
2. 改完递增 `?v=YYYYMMDD` 版本号
3. 重写 CSS 时先列出所有 class
4. 深色背景配色先确认强调色用浅雾蓝

### 涉及外部内容（课件/视频/他人文章）时

1. 开头加时效性声明
2. 用 WebSearch 核实最新版本信息
3. 人称动态处理
4. ASCII 图/终端块转莫兰迪 SVG

## 九、实战经验给下一个 Agent

### 标签：克制比丰富重要

每篇 3-4 个标签。核心概念的判断标准是"读者搜这个词时，这篇文章应该是他想要的结果"。颜色按父大类同色，4 色莫兰迪，同大类同色有图谱色族聚类的层次感。

### 深色背景配色：暖驼是浅色页专用

深色宇宙背景（星河主页浮层、技术 top 浮层）上的强调色用浅雾蓝 `#a8bdd1`，不要用暖驼。

### 课件搬运：时效性 + WebSearch 核实

模型版本、产品名、定价等易过时信息，**用 WebSearch 核实最新**。

### 改完一定要 build

改了 `posts/*.md` 后，**必须运行 `python build.py`**。改了 `js/articles.js`（配置）不需要 build，但要递增 HTML 里的版本号。

### Edit 工具对中文长句可能失败

Fallback 到 Python `content.find()` + index 切片替换。

### 工具页 CSS 多文件格式不一致

8 个工具子页面 CSS 结构相似但格式不同（有的单行有的多行）。批量修改时用 Python 脚本，每种格式都单独匹配。
