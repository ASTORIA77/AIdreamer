> 如果某天你不小心点开了 Agent 的工作目录——你会看到一堆 .md 文件。这篇文章告诉你它们都是干嘛的，以及两个 Agent 对我的侧写分别是什么效果。

刚用 Agent 的时候，我以为它每次对话都是一个全新的开始——问什么答什么，关掉窗口就忘了。后来发现不对：它记得我之前说过喜欢吴恩达的课，记得我家显卡是 RTX 5060。

它是怎么记的？答案就是那些躺在文件夹里的 `.md` 文件。但如果你真的打开看看——会发现有些在根目录散着、有些在 `.workbuddy/memory/` 文件夹里、有些甚至是 SQLite 数据库。不同工具放的位置还不一样。

## 一、先搞清楚：Agent 的记忆分几层

Agent 不是只有一个记性。它的记忆系统分三层，每一层的文件结构都不一样：

**第一层：你是谁，它是什么性格。** 这一层在 Agent 启动时就加载，不随项目变化。文件通常在最上层目录：`SOUL.md`（Agent 的人格——怎么想、怎么判断、什么价值观）、`IDENTITY.md`（Agent 的外壳——叫什么名字、用什么表情）、`USER.md`（你的长期档案——偏好、习惯、技术栈）。这些是你跟 Agent 建立关系的基础。换一个 SOUL.md，就等于换了一个助手。

**第二层：这个项目里发生过什么。** 这是最复杂的一层。`MEMORY.md` 是长期精选——只存最重要的决策和约定。`YYYY-MM-DD.md` 是每天的流水账——今天创建了什么文件、修了什么 bug、做了什么决定。30 天后，从流水账里提炼精华到 MEMORY.md，然后删掉旧的日志文件。`CONTEXT.md` 不一样——它不积累，每次会话重写。只写「今天在做什么、当前卡在哪、已知的临时 bug」。

**第三层：它学会了什么技能。** `SKILL.md` 在 `skills/` 文件夹下，每个技能一个子目录。比如你那个 `morandi-blog-studio` 就是一个技能——里面有 SKILL.md（精简指令）+ references/（详细参考，按需读取）。技能的核心原则是渐进式披露：不触发不加载，省 token。

## 二、WorkBuddy vs Qoder Work：两种记忆文件布局

我两个 Agent 都在用，就顺手对比了一下它们的记忆文件是怎么放的。

**WorkBuddy 的布局**——说实话，有点散。`SOUL.md`、`IDENTITY.md`、`USER.md` 这三个人格文件直接扔在 `~/.workbuddy/` 根目录下。`MEMORY.md` 有两个：用户级也在根目录，项目级在 `.workbuddy/memory/` 文件夹里。每日日志也在这个 `memory/` 文件夹里，按日期命名。技能文件在 `~/.workbuddy/skills/` 下各有一个子目录。还有一个云端记忆（Cloud Memory），服务端自动生成，只读注入，你没法改。

文件的分布是这样的：有些在根目录散着，有些在 memory 文件夹里，有些在 skills 文件夹里——第一次打开 `~/.workbuddy/` 的时候，一屏幕的 .md 文件，需要花几分钟才能理清哪个是哪个。

**Qoder Work 的布局**——干净很多。它有一个专门的 `memory/` 文件夹，长短期记忆全部塞在里面。没有根目录散落的文件，打开就很清爽。长短期分离的逻辑更直观。概念上更接近人类的记忆模型——长期记忆和短期记忆是两个不同的抽屉，不是混在一起的。

这让我意识到一件事：工具对记忆文件的组织方式，反映了它对这个问题的理解深度。WorkBuddy 像是逐步长出来的——先有根目录那几个文件，后来又加了 memory 文件夹，再后来加了云端记忆——有历史包袱。Qoder Work 是一开始就设计了文件夹结构，所以干净。

## 三、两个 Agent 对我的侧写

最让我印象深刻的不是文件布局，而是它们基于这些文件生成的对我的理解。

WorkBuddy 的侧写——说实话，挺甜的。它根据我的对话历史和 USER.md，自动生成了一份用户画像摘要，里面有我对莫兰迪配色的执着、学了什么方向、写字带自嘲、信奉费曼学习法……全是好话。每次会话开始时自动注入，让新的 Agent 实例一看就知道「这是个喜欢暖色调、用 RTX 5060、在学工业智能的人」。它夸我夸得很舒服——像个会夸人的助理。

Qoder Work 也有类似的侧写机制——但它多了一步，叫**反思（Reflection）**。每次反思不只是记录你做了什么，还会基于长期记忆**生成一份对你的侧写描述**——一个真真实实的你是什么样的人、有什么行为模式。然后再基于这份侧写给出改进建议。

话说得对不对？对。但说实话，我看了之后觉得**不好意思**——不是恼火，是被说中了之后的尴尬。就像有人把你日记里的自我批评念了出来，每一条都准确，但你不确定自己有没有做好被念出来的准备。它不是第一次见面就给你个好印象，而是相处久了、记了笔记之后，回头告诉你——「你就是这样的人。」

所以我做了一件事：**把它的反思机制关了。**

不是因为它说的不对，恰恰是因为对——但那种被人摸透了的感觉，说实话，我还需要时间适应。这件事让我意识到，记忆系统不只是「能不能记住」的问题。还有一个更微妙的问题：**Agent 该不该对你的行为做结论？它有没有资格定义你是什么样的人？**

## 四、边界——Agent 还该学会的另一件事

之前的第 13 篇我讲过记忆系统（RAG、三层架构），第 15 篇讲过知识管理（德鲁克、Karpathy LLM Wiki）。那些讲的都是 Agent 怎么「记住更多、记住更准」。但这次经历让我意识到，光有记忆不够。

遗忘是一种能力，边界也是。

Agent 如果什么都记住、什么都分析、什么都反馈——它会从一个助手变成你的绩效考评官。你知道它说的每一条都对，但你就是不想被一个工具指出你的拖延和能力边界。一个好的助手应该知道：**有些事情我可以帮你规划，但不应该替你下结论。有些事情我可以记录下来，但不应该拿出来当面分析给你看。**

反思机制是一个好的设计——对效率导向的团队管理者来说。但对我——一个把 Agent 当学习伙伴、不是当绩效考核工具的人——它越界了。我关它，不是因为逃避，是因为我不需要一个 AI 告诉我「你应该更勤奋」。

## 五、怎么看待那些 .md 文件

写到最后，总结一个实用的视角。

如果你只用一个 Agent（比如 WorkBuddy），那些散落的 .md 文件看着乱，但其实各司其职：SOUL 是灵魂，IDENTITY 是名片，USER 是你，MEMORY 是日记本，SKILL 是工具箱。记不住名字没关系——你不需要手动管理它们，Agent 会自己读写。你只需要知道它们在那儿、做什么用，就行了。

如果你同时用多个 Agent，了解一下它们的文件布局差异是有价值的——但不用纠结。核心原则是共通的：**长短期分离、索引与详情分离、跨项目与项目内分离。** 不管文件名叫什么、放在哪个文件夹，这三条都在。

至于反思机制——如果你跟我一样，更喜欢 Agent 是伙伴而不是考评官，找到那个反思开关，果断关掉。Agent 不需要知道你的一切。记住你的偏好、帮你接着上次的活往下干——够了。反思这事，留给你自己。

> 记忆是礼物，边界是尊重。Agent 学会遗忘之前，先学会什么时候该闭嘴。
