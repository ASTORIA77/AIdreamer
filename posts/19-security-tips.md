> 不是安全专家，就是个普通用户。但这几个习惯，我用着挺踏实。

## 沙盒隔离 + 默认权限：两条底线

我真正懂的就两条。

**第一条，沙盒隔离。** Agent 在我的电脑上跑，它可以读、可以写、可以操作文件——但如果它越界乱搞，沙盒能在操作系统层面拦住它。就像给它划了一个圈，圈内随便折腾，圈外的系统文件碰不到。大部分正经的 Agent 桌面端都有这个能力，区别只是隔离等级——从进程隔离到完整虚拟化，越深越安全。

**第二条，默认权限。** 别给 Agent 最大权限。安装软件的时候，它想要什么权限就给什么——那我就是在给它发通行证。我自己的习惯是：先按默认的来，最小权限。需要开放什么，等真正用到了再打开。不是给它发工资，不用一开始就把钥匙全交出去。

这两条不是什么高级操作。但守住它们，就不太容易出大事。

## 我就是按默认走的——以 WorkBuddy 为例

写这篇的时候我特意去翻了翻 WorkBuddy 的安全文档，发现它的默认设定其实比我想象的厚实。拆开看看：

**刚装完是个"白板"。** 默认零权限——不读任何文件、不连任何文件夹。得你自己在设置里手动勾选"允许访问"的目录，它才开始能碰。系统敏感目录（C:\Windows\System32、%APPDATA%）它自己就拦了，你想让它碰它都不碰。

**三层沙箱叠着。** 云端 Lighthouse 语义校验（识别高危指令，rm -rf 那种直接在云端拦截，连指令都不会下发到你电脑）+ 企业 iOA 零信任网关 + 本机 PC 管家沙箱隔离。三层叠起来，不是单靠一层在扛。

**高危操作有二次确认。** 删文件、批量改文件、跑脚本——这些都会弹窗让你手动确认。不是静默执行。而且每次操作都生成一个唯一 TraceID，记录时间、源 IP、调了什么技能、返回状态——日志不可删、不可绕，满足等保 2.0 三级审计。

**数据不出设备。** 文件处理全是本地的。需要调云端大模型推理的时候，只传必要的指令和最小上下文，不会把你整份 PDF 或 Excel 原封不动发上去。本地缓存用 AES-256 加密，密钥你本地生成，腾讯自己都没法解密。

**我唯一检查过的地方：** 去「设置 → 服务条款与数据权限」看了一下"匿名化输入输出用于模型优化"这个开关——默认是开的。你可以关掉，不影响任何功能。关了之后把诊断数据和自动错误报告也设成"从不发送"，基本就干净了。

说实话，这些我平时根本想不起来。就是装好之后按默认走，没动过。它已经帮你把该拦的拦了、该隔离的隔离了。我做的只是没主动关闭沙箱、没给它最大权限、没把涉密文件拖进去。

这就够了。说句大实话——上面这些术语我一大半都不太懂。什么零信任网关、等保 2.0、AES-256，写这篇文章的时候才现查的。但我不需要懂，因为它默认开着就行。

## 数据隐私：门槛在自己手上

Agent 安全这件事，技术层面的防线是软件开发商的责任。但有一道防线，在自己手上——**别把涉密文件给它。**

这不是技术问题，是意识问题。你跟 Claude 聊项目方案、让 DeepSeek 润色报告——这些对话和文件，是发到了它们的服务器上的。如果文件涉密，这一步就已经把数据交出去了。

我自己有个简单的判断标准：**这个东西敢不敢发到公开的微信群里？敢，就给它。不敢，就别给。** 不用想太多。

## 别听它自己说——看权威机构怎么说

每一个 Agent 都会讲自己的安全防卫有多厉害。几层加密、什么级别的隔离、零信任架构……听起来都挺唬人的。

但我信的不是它们自己说的。我信的是**中国信通院的评估**。

2026 年初，中国信通院启动了智能助理智能体（Claw）可信能力评估，覆盖 5 大检验项目、57 个能力项——平台可信、供应链可信、权限管理、数据管理、运行透明，都有具体指标。不是厂商自说自话，是第三方权威机构按统一标准来测。达到要求的，给评级，最高的 4+ 级。

比如腾讯 WorkBuddy V4.10.0，我用的就是这个——它通过了首批评估，在沙箱隔离、供应链审查、权限管控、数据加密这些维度都过了。不是我看了宣传就觉得安全，是信通院测过了。

这个逻辑很简单：**软件说自己安全我不信，政府背书的第三方评估我信。** 就像你买食品不看包装上的广告，看质检标志。一个道理。

## 就这些

我不是安全专家，这也不是什么安全教程。就是一个普通用户用 Agent 时留下来的三个习惯：

1. **守好沙盒和权限。** 别一开始就把家门全打开。
2. **别把涉密数据给它。** 敢发微信群的再给 AI。
3. **看权威评估，别看自吹。** 中国信通院的 Claw 可信能力评估是个好东西，别管一个软件说自己多安全，谁测过信谁。

就三条。用完晚上睡得着觉。
