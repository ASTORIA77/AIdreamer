> CLI 是给机器和懂机器的人用的；GUI 是给所有人用的。Agent 时代，两者不是谁替代谁——是各管各的。

## 一、先搞清楚：CLI 是什么

CLI 就是 **Command Line Interface**——命令行界面。你在屏幕上看到的是一个黑底白字的窗口，敲一行命令，机器执行，返回结果。没有按钮，没有拖拽，没有窗口。

与之相对的就是 GUI——**Graphical User Interface**，图形用户界面。你在屏幕上看到的是窗口、图标、菜单、按钮。你用鼠标点、用手指划——这是绝大多数人跟电脑打交道的方式。

CLI 是计算机最早期的交互方式。在图形界面出现之前，所有人跟电脑交流都得靠命令行——你敲一行，它回一行。后来有了 GUI，大多数人不再需要碰命令行了，但开发者、运维工程师、系统管理员一直在用。为什么？因为 CLI 有几个 GUI 做不到的优势：

- **快。** 一行命令能干完 GUI 里要点七八下的事。
- **可组合。** 命令之间可以用管道（pipe）串起来，前一个命令的输出直接喂给后一个命令。
- **可脚本化。** 你可以把一串命令写成脚本，让它自动跑，不需要人坐在前面点。
- **可远程。** 服务器通常没有屏幕，你通过 SSH 连上去，就是命令行。

换句话说，CLI 是开发者跟机器"高效对话"的方式——精炼、精确、可编程。

## 二、GUI 的诞生：给人类看的界面

GUI 的历史比很多人想象的早。

1968 年，Douglas Engelbart 做了著名的"Mother of All Demos"——第一次展示了鼠标、窗口、超链接、文字编辑。1973 年，Xerox PARC 推出了 Alto——第一台带图形界面的个人电脑。但这些都没走向大众。

真正让 GUI 普及的是 1984 年的 Apple Macintosh——"1984" 那支广告砸下来，个人电脑从此有了"面孔"。1990 年代微软 Windows 的崛起，把 GUI 带到了全球每一台办公桌上。

GUI 带来的最深刻的变化不是"好看"，而是**功能模块化**。每一个按钮代表一个功能，每一个窗口代表一个任务，每一个菜单代表一个分类。你在使用 GUI 的过程中，潜移默化地学会了把复杂操作拆成一步一步——这恰好就是后来软件工程里"模块化设计"思想的源头。现代软件开发的组件化、微服务化，追到底层逻辑，跟你在 GUI 里点按钮选菜单是同一个思路：**把大东西拆成小东西，各自独立，组合使用。**

## 三、为什么 Agent 天生适合跑在 CLI 里

到了 AI Agent 时代，CLI 突然变成了一个热门话题。原因很简单——**Agent 的底层操作方式跟 CLI 是同构的。**

Agent 是怎么干活的？它通过 Bash、PowerShell、Shell 这些命令行工具来操控你的电脑——读文件、写代码、跑脚本、装依赖、提交 Git。这些操作天然就是命令行指令。Agent 不需要"点按钮"——它直接敲命令，速度比人快几个数量级。

这就是为什么现在最火的 Agent 工具都选择以 CLI 为主界面：

**Claude Code** 是 Anthropic 出的命令行 Agent。你在终端里跟它说话，它直接在你的项目目录里读写文件、跑测试、提交代码。没有 GUI——一个终端窗口就是全部。它的设计理念是：Agent 跑在终端里，就像开发者跑在终端里一样自然。

**飞书 lark-cli** 是 2026 年 3 月飞书官方开源的命令行工具，MIT 协议。它把飞书的 2500+ API 封装成了 200+ 命令——消息、文档、日历、多维表格、邮箱、任务、会议全覆盖。Agent 通过执行 Shell 命令就能帮你发消息、查日历、读文档。上线两天 GitHub 星标突破 4000。

越来越多的软件开始开放 CLI——GitHub、GitLab、Notion、Linear。这不是赶时髦。这是它们意识到：**在 Agent 时代，你的软件不仅要给人用，还要给 Agent 用。而 Agent 最自然的操作方式，就是命令行。**

GUI 是给人看的——你需要用眼睛找按钮、用鼠标点。CLI 是给机器用的——Agent 直接敲命令，不需要看，不需要找，不需要等界面渲染。在 Agent 的世界里，CLI 的效率碾压 GUI。

## 四、但 GUI 不会消失

说了这么多 CLI 的好，我现在要讲另一面。

**只要人还在操控电脑，GUI 就会一直在。**

为什么？因为 GUI 就是给人看的。人不是机器——你需要在屏幕上看到东西才能理解。你需要用鼠标选一段文字才知道"这段我要改"。你需要看到文件的缩略图才知道"是这个不是那个"。你需要拖拽、缩放、对比——这些操作天然是视觉的。

我自己用 Agent 的时候，WorkBuddy 给我提供了一个很舒服的 GUI——我输入文字、看它返回的结果、看它画的图、看它生成的 HTML。我不需要开一个终端敲命令。**我为什么非得让自己不舒服呢？**

CLI 适合 Agent，但 Agent 的使用者不只有开发者。一个做市场的人、一个做运营的人、一个学 AI 的学生——他们不需要学会用终端才能用 Agent。GUI 降低了使用门槛，让更多的人能用上 AI——这件事本身就有巨大的价值。

所以我的判断很简单：**CLI 是 Agent 的跑道，GUI 是人的客厅。** Agent 在 CLI 里跑得最快最稳，但人坐在 GUI 前面最舒服。两者不矛盾——Agent 在后台跑命令行，你在前台看图形界面。好的产品就是那个把两者连起来的翻译官。

## 五、"SaaS 已死"——真的吗？

2026 年初，一篇叫「SaaS 已死，SaaS 永生」的文章在圈里引发了很大争议。核心观点是：Agent 时代来了，软件的使用主体从人变成了机器，传统 SaaS 的 GUI 界面将被剥掉——变成所谓的"无头 SaaS（Headless SaaS）"，只留底层 API 给 Agent 调用。文章管这叫"有头守人性、无头喂 AI"。

这个说法有一定道理——Agent 确实不需要点按钮，它只需要 API 和命令。但"SaaS 已死"这个标题，我觉得太绝对了。

我的看法很朴素：**只要人还在用电脑，GUI 就不会消失。** SaaS 是给人用的——尤其那些沉淀了领域知识（Domain Knowledge）的 SaaS，比如财务系统、ERP、审批流。这些东西的核心价值不在界面，而在于里面积累的行业规则、业务逻辑、合规要求。Agent 可以帮你调数据、走流程，但最终的决策、风控、签字——还是得人来。

虎嗅那篇文章自己也说了：Agent 能自动整理数据、同步台账、发起流程，但遇到异常场景、合规核查、跨部门冲突时，依旧需要人类介入判断、兜底拍板。而有头 SaaS 的 UI，就是人类守住商业确定性的窗口。

所以真正会发生的不是"GUI 消失"，而是**分工**：无头的那部分给 Agent 跑，有头的那部分给人看。日常的机械操作交给 Agent，关键决策的查看和审批留给人。

至于什么时候 GUI 会真的消失？可能得等到人不再工作了——不用看屏幕了、不用审批了、不用做决策了。那会是什么时候？谁知道呢。未来什么样，我不急着下判断。

自己用的 WorkBuddy 就是这个逻辑。你看到的界面是 GUI——对话框、文件列表、任务面板。但 Agent 在后台执行操作时，走的是命令行和 API。你不需要知道它敲了什么命令，你只需要告诉它你要什么，它自己干。

这不是退而求其次。这是分工——**机器干机器擅长的，人干人擅长的。** CLI 不会消灭 GUI，GUI 也不会消灭 CLI。它们会像汽车的自动挡和手动挡一样长期共存——大多数人开自动挡，但赛道上永远是手动挡更快。

## 六、计算机的世界是人造的

写到这里，我突然想到一个更大的感受。

学 AI 学到现在，连带补了不少计算机的知识。最大的感触不是"技术好难"——而是**这个世界真奇妙**。

它跟自然界不一样。自然界的规则是"被发现"的——万有引力在人类来到之前就在了。但计算机世界的规则是"被发明的"——从二进制到操作系统，从 CLI 到 GUI，从编程语言到 SaaS 的商业模式，每一条规则都是某个人或某一群人在某个时间点定下来的。

这意味着什么？意味着这个世界的物理定律是可以变的。

Flash 就是最好的例子。我小时候，Flash 是互联网的国王——网页动画、小游戏、视频播放器几乎全是它。巅峰时期 Web 端使用率超过 80%。然后 2020 年 12 月，Adobe 正式停止支持，2021 年 1 月起全面禁用。从 80% 到被彻底停用，就几年时间。一个统治整个互联网的技术，消失得干干净净。这不是自然淘汰——是整个生态决定换一套规则。

编程语言也一样。Pascal 曾是一代人的编程入门课，Delphi 是 Windows 桌面开发的王者，Perl 是 1990 年代的脚本之王，Ruby 是人气偶像。它们都是各自时代"最好的选择"——然后在下一个时代被更好的选择替代。不是它们错了，是时代变了，约束变了，它们就退场了。

所以 CLI 和 GUI 的争论，放在这个大背景下看——其实没有标准答案。今天 CLI 吃香，是因为 Agent 需要它。明天如果 GUI 进化出了更适合人机协作的形式，CLI 可能又退回到开发者的工具箱里。但有一件事不会变：**只要你理解了规则是怎么被设计出来的，你就不再害怕规则会变。** 因为新的规则也是人定的，而你也是人。

学 AI 也是一样。今天的框架明天可能就换了，大模型明年可能就被新的取代。但你攒下来的——对系统的理解、对抽象的感知、对"人怎么用机器解决问题"的直觉——这些东西不会过时。

计算机的世界是人造的。而你在学的东西，本质上是理解一群人怎么造了一个世界。
