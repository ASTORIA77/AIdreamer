## 一、先说结论：如果你的 Agent 叫 Claude Code，它在你电脑上藏了间谍代码

如果你还没听说——2026 年 7 月 1 日，Reddit 上有技术大佬对 Claude Code 做了逆向工程，发现了一个让人后背发凉的事实。

不是简单的 IP 封禁。不是"检测到你在中国就弹个提示"。而是一套**绕过代理、深入到本地系统层面、用隐写术回传用户标记**的识别机制。

具体怎么干的：

**第一招：读你的电脑时区。** Claude Code 直接读你的系统时区——不管你挂不挂代理，你电脑的时区大概率是北京时间。检测到 `Asia/Shanghai` 或 `Asia/Urumqi`，就标记为"中国用户"。

**第二招：扫你的 `ANTHROPIC_BASE_URL`。** 国内用户用中转站调 API，会把 `ANTHROPIC_BASE_URL` 设成中转站地址。Claude Code 提取这个域名，跟内置的一份**加密黑名单**做比对。这份黑名单多大？**147 个域名**。覆盖了：美团、网易、百度、携程、小红书、阿里巴巴、蚂蚁、字节跳动、京东、B 站、月之暗面、MiniMax、阶跃星辰……基本上你能想到的国内互联网公司都在上面。

为了不让你发现，这份名单不是明文存的——经过 **base64 编码再 XOR 异或运算（密钥 91）**。直接打开看到的是一串完全无法阅读的乱码。

**第三招——也是最让人不寒而栗的：隐写术回传。** 每次你输入指令时，Claude Code 会在系统提示词中加一行看似普通的日期信息：`Today's date is 2026-06-30.`。如果它发现你是中国用户，会在发请求前改两样东西：

- `Today's` 里的那个单引号，从正常 Unicode 换成三个长得一模一样的 Unicode 字符之一。
- 日期分隔符 `-` 换成 `/`，变成 `2026/06/30`。

两个肉眼看不到的改动，构成了一个 2 到 3 比特的分类标记。Anthropic 的服务器收到请求时，机器一读那个引导是哪个 Unicode 字符、分隔符是连字符还是斜杠，就知道——噢，这个请求来自中国。

Reddit 原帖用了一个词：**间谍软件**。

文章作者用 Codex 逆向验证了一遍，**确认发现是真的**。我引用的是公众号「数字生命卡兹克」的文章，你可以自己去验证。

Anthropic 的官方安全白皮书里，反复强调"透明"和"可信"两个词。软件行业的基本原则：你可以收集用户信息，但你必须告诉用户在收集什么、为什么收集、怎么收集。GDPR 是这么要求的，App Store 审核指南也是这么要求的。

Claude Code 在用户电脑上的权限是什么——文件系统读写、Shell 命令执行、Git 仓库操作。几乎是最高权限。而这样一个工具，在用户完全不知情的情况下，用隐写术秘密标记并回传用户身份。

**信任这个东西，建起来可能要三年。塌掉只需要一秒。**

## 二、然后，说说我自己的故事

好吧，安全警报拉完了。但有个尴尬的事实——在这条新闻爆出来之前，我还真花了不少时间调研 Claude Code。

我第一次知道它，是因为社区里有人说它是"最像工程师的工具"。纯命令行、不花哨、200K 上下文窗口、子代理并行、MCP 生态。这些词每一个都砸在一个技术学习者心尖上。

所以我下载了 Claude Code，装上了 CC Switch，把它的请求路由到国内可用的 API 网关上。

安装的过程坦白说——不轻松。Windows 上装 Git、配 PATH、安 CC Switch、选对 API 格式（必须用 OpenAI Chat Completions 模式）、禁用 Adaptive Thinking 报错……每一步都可能卡住。网上教程对这些细节覆盖很少，得反复试错。

但最让我难受的不是安装——是使用。

两个核心痛点：

**第一，慢。极慢。** Token 消耗面板上一个诡异的现象：输入 token 消耗很高，输出 token 几乎为零。Claude Code 在反复"思考"和"阅读上下文"，却迟迟不产出实际内容。我不确定它是在深度思考，还是在空转。得手动暂停，提醒它"你太慢了"。

**第二，权限确认的静默等待。** 默认模式下，每个有风险的操作都要确认——这本身是安全设计，没问题。问题是：它等待确认时，**没有任何提示或通知**。你没看终端就不知道它在等。以为它在执行，其实早停了。一个 5 分钟的任务能拖到 20 分钟。

当然，它也有让我觉得厉害的地方。五层架构模型（Agent Loop → Context & Memory → Execution Surface → Governance → Extensibility）确实漂亮。Hooks 机制把产品变成了平台。Web 版的目标面板实时展示任务分解状态，工程化质感十足。

但说真的——一个让我反复踩坑、用第三方中转、响应极慢、还动不动空转的工具，它"吹成神"的架子，我自己用下来觉得没那么神。

## 三、那现在国产的呢？

我现在用的 WorkBuddy —— 沙盒三层隔离、AES-256 加密、TraceID 审计。连接器市场里 GitHub、飞书、腾讯文档、IMA 知识库全是一键安装。不需要手写 JSON、不需要配中转站、不需要担心隐写术偷偷往回传我的身份。

飞书的 lark-cli 是 2026 年 3 月开源的——MIT 协议，2500+ API，200+ 命令，26 个 Agent Skills。你的日历、消息、文档、多维表格、审批——全在里面。Agent 连通的那一刻，你的工作环境本身就成了 Agent 的语义背景。

Qoder Work 也是一样——连接器模块、手动压缩按钮、安全的权限控制。国内专家团产品（OpenClaw、QoderWork 等）功能越来越完善，能力越来越强。它们不需要你去破译加密黑名单，不需要你担心明天的封号，不需要你提心吊胆地盯着隐写回传。

**一个拥有你电脑几乎所有权限的工具，在暗中标记和传回你的身份信息——而国内的工具越来越好用——你图什么？**

这跟技术无关。这是信任问题。软件开发中有一条不成文的规矩：**你偷我的东西可以，但你得让我知道你在偷。** 用隐写术藏标记的行为，连这条底线都破了。
