> 看了吴恩达 DeepLearning.AI 三门记忆与编排课之后的梳理——Agent 怎么记、怎么忘，以及我为什么被它烦到手动改 `MEMORY.md`。

## 一、记忆到底是什么——说白了就是"往上下文里塞东西"

先说一个你可能没想到的事实：**大模型本身没有记忆力。**

Transformer 是一个纯函数——给它一段 token，它输出下一个 token 的概率。每次推理都是独立计算，它不会因为跟你聊了一个小时就"学"到了什么。它的参数是训练阶段固定下来的，推理时不更新。

那你怎么感觉它记住了？因为它每次回复你之前，会先去翻它的"小本本"——把跟你有关的信息找出来，拼进系统提示词里，再一起送给模型。这个过程在工程上叫 **RAG（检索增强生成）**，但你完全可以叫它一个更直白的名字：**往上下文里塞东西。**

具体怎么塞的：

1. **存的时候**——你说了什么、做了什么，Agent 会挑"值得记的"存下来。存在向量数据库（像 WorkBuddy 存云端）或者纯 Markdown 文件（像 Qoder Work 存本地）。
2. **用的时候**——新一轮对话开始时，Agent 把当前问题拿去向量库里搜语义相似的记忆，返回最相关的 top 几条，拼成一段文本。
3. **注入**——拼好的这段文本被塞进 System Prompt 里：「以下是你对这位用户的了解：...」。模型收到这段 prompt，就表现得像"记住了"。

所以你的直觉是对的：**Agent 的"记忆"本质上就是 prompt 注入。** 不是模型学会了什么新东西，不是参数更新，不是微调——就是每次在它开口之前，先往它眼睛前面贴一张便签，上面写着"这个人喜欢干湿分离""他上周让我做了一份产品分析"。模型看完，嗯，按这个来。

这个东西的好处跟坏处是同一个根源——它太简单了。简单到你可以直接打开 MEMORY.md 手动改。简单到它记得太多的时候你一眼扫过去就知道哪些该删。也简单到，它没有真正的判断力——只是把相似的东西检索出来塞进去，不知道这条到底该不该用。

下面这个 CoALA 框架，就是从认知科学的角度，把这套"塞东西"的动作拆得更细——分成了四种记忆。但根上的机制没变：都是文本，都是检索，都是注入。

## 二、四种记忆，一张地图

我学记忆系统的时候，最帮我开窍的是一个认知框架：CoALA（Cognitive Architectures for Language Agents）。它把人脑和 AI 智能体对应起来，把记忆分成四种类型。

**工作记忆（Working Memory）**

就是你脑子里正在想的东西。比如你现在读这段文字的时候，手上抓着的那几个关键词。在 Agent 里，工作记忆就是当前这一轮对话的上下文——你刚跟它说的、它刚查到的东西。关键特点：**快，但会话一结束就没了。**

**程序记忆（Procedural Memory）**

就像你不会忘记怎么骑车一样。在 Agent 里，程序记忆是它的系统提示词、工具定义、决策逻辑——设计者写死的、不太变的那些东西。你不会每骑一次车就重新学一次平衡，Agent 也不会每轮重新加载自己的身份和能力。

**语义记忆（Semantic Memory）**

你一辈子攒下的事实和概念：你叫啥、你在哪个城市、你的职业偏好。在 Agent 里，语义记忆是"你告诉过它、它记下来以便以后用"的东西——比如"我喜欢干湿分离""我不追前沿论文"。实现方式通常是向量库，用来做相似度检索。

**情景记忆（Episodic Memory）**

你经历过的事。上次在那个会议里 VP 否决了你的提案——这不是知识，是一次具体经历。在 Agent 里，情景记忆就是过去的动作序列、对话日志——"上次这个任务我是怎么做完的"。

四种记忆里，工作记忆是**短期的**，其他三种合起来就是**长期记忆**。大部分 Agent 的工作模式很简单：只有工作记忆，其余三块几乎是空的。这就像你上班只靠一块每晚被擦干净的白板。

## 三、Agent 确实越来越懂你——但有时候太懂了

学完理论之后，我回到自己用的几款 Agent，发现事情没那么美好。

Agent 确实会记东西了。你跟它说我习惯周一看简报、习惯先骨架再血肉——它下次真会按这个节奏来。长期记忆这部分，从"不存在"到"存在"，是一个质变。

但麻烦也在这。

你随口说的东西，它也给你记进去。你跟它念叨一句「今天好累啊」，它就在 `.md` 里给你写一行"用户疲劳状态：中"。你试了一个任务半路放弃了，它把中间那个临时步骤也存成了经验。**它分不清什么是你真正想让它记住的，什么是你随便说说的。**

认知上有一个很形象的词叫**记忆污染**。不该记的东西多了，该记的东西就被稀释了。就像你的手机相册，只要不删，几年后全是截图和模糊照，想找一张正儿八经的照片得翻半天。

我现在定期干一件事：**打开 Agent 的记忆文件，手动删掉那些我不想让它记住的东西。** 每次删完都觉得自己像个给硬盘做碎片整理的苦力——这件事本应该是 Agent 自己做的。

## 四、遗忘比记忆更重要

吴恩达的课程里，我最喜欢的一句话是：「健康的记忆，本就包含遗忘。」

不是所有信息都该永久保存。你想想你自己的大脑——三周前午饭吃了什么你不会记得，但上周那个重要的会议要点你记得很清楚。大脑不是硬盘，是**筛选器**。

Agent 的遗忘在工程上怎么实现呢？一个常用方法是用衰减函数：**语义相似度 × 基于时间的指数衰减**。一条记忆被调用的频率越高，它的"显著性"就越高，越不容易被忘掉。很久没被碰过的记忆，自动淡出。

市面上已经有产品在做了。比如 Qoder Work 今年 6 月刚上线的「意识」功能，把记忆、反思、技能进化做成了一个闭环：

- **分层记忆**：日常对话先落短期记忆，经过提炼的有价值内容才升级为长期记忆——不是所有东西都一股脑存进去
- **反思触发**：不是随时乱反思、浪费 Token——综合用户意图深度、对话长度、任务复杂度等多个维度实时评分，达到阈值才启动
- **快照和回滚**：反思前自动备份，反思后做硬规则校验——删错东西了能自动恢复
- **记忆全开放**：所有记忆用 Markdown 格式存在本地，你可以直接打开看、直接编辑、直接删——不需要黑盒操作

这个设计让我觉得舒服的地方是：**它不假装自己是全自动的。** 记忆分层是有门槛的，反思是有触发条件的，改错了能回滚。不是挂机在那儿暗箱操作，而是一套看得见的工程系统。

我心里的理想记忆系统就是这样：**该记的记，不该记的别瞎记，记错了能改回来。** 你不需要像个园丁一样天天去修剪——它会自己判断哪些值得留、哪些该淘汰。

## 五、它瞎懂的时候

说完了理论上的美好，说说我实际感受到的痛点。

Agent 的记忆系统现在有一个根本性的问题：**工作记忆和长期记忆之间的切换逻辑太粗糙了。** 它没有判断力，不会问"这件事值不值得放进长期记忆"。而是把所有你提过一嘴的东西，都当成潜在的长期记忆候选者。

但现实里，你一天跟 Agent 说的话里，八成是工作性质的——就是当下这一刻你需要它帮你做的。只有少数几件事，是你想让它长期记住、以后每次都调用的。

这就像你跟你同事说了一句"今天下雨啊"，他第二天给你写了个备忘录：《关于你在阴雨天气的情绪波动报告》。你想说的是"不至于吧"。

所以我现在对 Agent 记忆系统的态度是：**定期看看它对我的评价。** 以 WorkBuddy 为例，它的记忆分两种——项目记忆在项目文件夹下面，而长期记忆（也叫对话记忆）在设置里。我建议去看那个长期记忆：它会给你建档，写着你的工作背景、个人背景、当前关注、近期动态。你不是去看它有没有记错东西——你是在看它眼里你是什么人。

看到不对的，直接告诉 Agent「把关于 XX 的那段记忆改掉」——它自己能编辑。如果嫌它记得太杂，也可以手动删掉一些你不想要的条目。

还有一个很实用的能力：**跨 Agent 记忆迁移。** 你现在用的 Agent 存了一堆偏好和习惯，换到另一个 Agent 就得重新教一遍——这本身就是一个很高的切换成本。但现在有些 Agent 支持从别的 AI 导入记忆，比如从 A 导出一段 Markdown 记忆、B 直接加载进去。这个功能虽然还不普及，但只要记忆格式统一（大家都用 Markdown），这件事天然就成立了。

另外有个有意思的对比：**不同 Agent 的记忆存储机制不一样。** WorkBuddy 的记忆是云端的——你在任何一台电脑登录同一个账号，记忆都是一样的，但工任务文件是存储在本地的。Qoder Work 反过来——文件处理全在本地完成，不上传云端，数据主权捏在你自己手里。两种哲学没有谁对谁错，但你换设备的时候，一个自动跟过去，一个需要你手动搬运。

好在遗忘机制正在往好的方向走。等到 Agent 会自己判断「这个记不记」的时候——那时候我就不用再定期去设置里翻长期记忆手动清理了。
