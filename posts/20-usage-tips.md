前面写了模型的选型、API 的细节、提示词的取舍、Skill 的协议层——该拆的都拆了，该掰的都掰了。但如果现在有人问我"用 Agent 最重要的心得是什么"，我的回答可能跟上面这一切都没太大关系。

## 一、体感是唯一的货币

聊使用方法论的帖子到处都有：根据不同的使用场景切换不同的模型，先打骨架再填充血肉，搭一套固定的工作流程，遇到什么任务就调用对应的专家 Skill，能用 MCP 自动化就别手动——这些不是没用，但这些都是别人的使用心得。

别人的使用心得也许能帮你一阵子。但你对一个技术真正的理解——那种不需要翻笔记就能直接判断"这个场景应该用这个工具"的直觉——只会在你亲手用过之后，一点一点长出来。

这不是故弄玄虚。想想你怎么学会用 QQ 的。你读的是产品说明书吗？你是每天发消息、传文件、拉群聊，手指头自己就记住了。没有人专门"练"过 QQ，但没有人不会用 QQ。

Agent 的道理一模一样。你用不用"骨架优先"的工作流，你用 WorkBuddy 还是 Qoder Work，你装了几个 MCP 连接器——这些选择本身不是心得。真正的心得是：你在某一次任务里发现某个模型对某种问题特别顺手，你在某一次失败后意识到某个工具的边界在哪，你在反复试过之后，手指头知道下一步该点哪里了。

**技术体感不是信息，是记忆。** 信息可以复制，记忆只能自己生成。

## 二、无非是个软件

还有一个拦路虎，叫**心理门槛**。

很多人不是不会用，是不敢用。"这个配置好复杂""万一跑崩了怎么办""我看不懂那个报错"——这些东西翻译过来其实就一句话：**你把它当回事了。**

但你想想，你用 QQ 的时候会紧张吗？你用微信传文件的时候会想"万一传失败了怎么办"吗？不会。因为你知道它就是个软件。消息发错了撤回，文件传失败了重传，大不了关掉重新打开。

Agent 没有什么不一样。它也是一个软件，跑在你的电脑里、你的浏览器里、你的终端里。任务崩了——再来一个。对话进程被取消了——重新发起。MCP 连不上了——排查一下，或者暂时不用，又不是世界末日。

数字世界的所有东西都有一个共通的属性：**可恢复**。你操作的不是实体设备，不存在"按错一个键整台机器就报废了"。最坏的情况是重来一遍——而重来一遍在数字世界里几乎不花成本。

这不是什么"心态建设"或者"抗压能力"。这就是事实。Agent 也好，API 也好，MCP 也好，它们在物理意义上跟你用了十年的 QQ 和微信一样——都是一堆代码在帮你干活。代码不会因为你点错了就记仇，不会因为你没看文档就罢工。它只会按照它被设计好的方式运行，理解不了就报个错，你改一下就行了。

## 三、只有你用过的，才是你的

绕了一大圈，其实就一句话。

不要等别人告诉你该怎么用，不要怕用不好。一切发生在你的电脑里，在数字世界里——最坏的结果无非是关掉窗口重新打开。

而最好的结果——你对一个工具的体感，对一种工作流的直觉，对自己"用 AI 学 AI"这套方法的自信——这些东西别人给不了，只能自己攒。

使用心得从来不是方法论。**使用心得是体感。体感是你和工具之间的私事。** 别人说得再好，那也不是你手的温度。
